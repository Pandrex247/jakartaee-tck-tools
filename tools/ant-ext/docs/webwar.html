webwar

Description:

This task is designed to support building of web .war files.

<webwar> will parse the deployment descriptor to determine the necessary class files. These files are assembled along with the deployment descriptor into an web .war file. Any support files which need to be included in the generated jar can be added with the <support> nested element.

Parameters:

Attribute

Description

Required

descriptor

xml descriptor.

Yes

name

Controls the naming convention used to name generated WEB wars. 

Yes
manifest
 use it only if the war needs custom manifest file
No
autocheckexcludes
 files not to be automatically included into archives
No

Nested Elements

The webwar task provides the following nested elements.

support

The <support> nested element is used to supply additional classes (files) to be included in the generated wars.  DO NOT use support for servlet classes that are specified in web descriptor.  The <support> element is a FileSet, so it can either reference a fileset declared elsewhere or it can be defined in-place with the appropriate <include> and <exclude> nested elements. The files in the support fileset are added into the generated WEB jar in the same relative location as their location within the support fileset. Note that when webwar generates more than one jar file, the support files are added to each one.


Content (for war's that need content)

The <content> nested element is used to supply web content (e.g., *.html, *.jsp, *.tld, etc) to be included in the generated wars.  You do not need to specify content for war's that do not need content.  Use multiple content subelements if the war takes content from multiple locations.

Examples

Example 1:
==========
<target name="package">
    <package>

	  <webwar descriptor="jsp_tagext_LifeCycle_web.xml" name="jsp_tagext_LifeCycle">
	  <support includes="tags/Count.class, tags/ATag.class, tags/TagSupport/DefaultReturnValues.class, tags/BTag.class, tags/misc/PersistenceTag.class, tags/SimpleIteratorTag.class, tags/SillyArgsExtraInfo.class, tags/Verbatim.class, tags/CheckCount.class,	tags/SillyArgs.class, tags/TagSupport/FirstPrizeTag.class, tags/misc/ReqTimeAttributesTag.class, tags/TagSupport/SecondPrizeTag.class"/>
	       <content dir="contentRoot"/>
	     <!-- includes all files	under contentRoot -->
	  </webwar>

      <appeear name="jsp_tagext_LifeCycle"/>
	   </package>
	 </target>


Example 2:
==========
  <target name="package">
    <package>

	    <webwar descriptor="jsp_deploy_enventry_single_web.xml" name="jsp_deploy_enventry_single">
	<support includes="com/sun/ts/tests/common/web/JSPBeanWrapper.class,	com/sun/ts/tests/common/web/WebUtil.class, com/sun/ts/tests/assembly/util/shared/enventry/single/TestCode.class, com/sun/ts/tests/jsp/ee/deploy/enventry/single/JSPBean.class"/>
	       <content dir="webFiles"/>
	  <content dir="." includes="*.jsp"/>
	     </webwar>

	  <clientjar	descriptor="jsp_deploy_enventry_single_client.xml" name="jsp_deploy_enventry_single"
	mainclass="com.sun.ts.tests.jsp.ee.deploy.enventry.single.Client">
	 <support includes="com/sun/ts/tests/common/web/WebServer.class"/>
	  </clientjar>
      <appear name="jsp_deploy_enventry_single"/>
    </package>
  </target>


Example	3:
==========
  <target name="package">
	   <package>
	    <webwar descriptor="servlet_jsh_HttpServlet_servlet.xml"	name="servlet_jsh_HttpServlet_servlet">
	      <support includes="tests/javax_servlet_http/HttpServlet/HttpServletCoreServletTest.class"/>
	 </webwar>
<!-- no content subelement -->
	  <appear name="servlet_jsh_HttpServlet"/>
	</package>
  </target>
