resrar

Description:

This task is designed to support building of resource archive.  By default, resrar includes ${ts.home}/dist/${pkg.dir}/*.jar and ${ts.home}/dist/${pkg.dir}/*.war.  If certains archives under ${ts.home}/dist need to be explicitly included or excluded, use support sub-element.

Parameters:

Attribute

Description

Required

name

Controls the naming convention used to name generated rar. 

Yes

descriptor

xml descriptor

Yes

Nested Elements

The <resrar> task provides the following nested elements.

support

The <support> nested element is used to supply additional component archives to be included in the generated rars. The <support> element is a FileSet, so it can either reference a fileset declared elsewhere or it can be defined in-place with the appropriate <include> and <exclude> nested elements. 

Examples

Example 1:
==========
  <target name="package">
    <package>
	<jar	jarfile="${dist.dir}/${pkg.dir}/whitebox.jar" basedir="${class.dir}"
	includes="${pkg.dir}/**/*, com/sun/ts/tests/connector/ee/util/**/*"/>
	  <resrar name="whitebox-xa-param" descriptor="ra-xa.xml"/>
	   <resrar name="whitebox-notx-param" descriptor="ra-notx.xml"/>
	<resrar name="whitebox-tx-param" descriptor="ra-tx.xml"/>
	 <resrar name="whitebox-xa" descriptor="ra-xa.xml"/>
	    <resrar name="whitebox-notx" descriptor="ra-notx.xml"/>
	   <resrar name="whitebox-tx" descriptor="ra-tx.xml"/>
    </package>
  </target>
