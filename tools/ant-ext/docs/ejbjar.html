ejb-jar

Description:

This task is designed to support building of EJB2.1 jar files.

ejb-jar will parse the deployment descriptor to determine the necessary class files which implement the bean. These files are assembled along with the deployment descriptor into an EJB jar file. Any support files which need to be included in the generated jar can be added with the <support> nested element. For each class included in the jar, ejb-jar will scan for any super classes or super interfaces. These will be added to the generated jar.

The jar files are only built if they are out of date. Each deployment tool element will examine its target jar file and determine if it is out of date with respect to the class files and deployment descriptors that make up the bean.

Parameters:

Attribute

Description
Required

descriptor

 xml descriptor for the ejb jar
Yes

name

Controls the naming convention used to name generated EJB jars.
Yes
manifest
 use it only if the jar needs custom manifest file
No
autocheckexcludes
 files not to be automatically included into archives
No

Nested Elements

In addition to the vendor specific nested elements, the ejb-jar task provides three nested elements.

support

The <support> nested element is used to supply additional classes (files) to be included in the generated jars.  DO NOT use support for ejb classes and interfaces that are specified in descriptor.  The <support> element is a FileSet, so it can either reference a fileset declared elsewhere or it can be defined in-place with the appropriate <include> and <exclude> nested elements. The files in the support fileset are added into the generated EJB jar in the same relative location as their location within the support fileset. Note that when ejb-jar generates more than one jar file, the support files are added to each one..

Examples

Example 1:
==========
  <target name="package">
    <package>

	    <ejb-jar	descriptor="ejb_sam_twobean_ejb.xml" name="ejb_sam_twobean" manifest="MY_MANIFEST.MF">
	   </ejb-jar>

	 <clientjar descriptor="ejb_sam_twobean_client.xml" name="ejb_sam_twobean"
	 mainclass="com.sun.ts.tests.samples.ejb.ee.twobean.TwoBeanClient">
	</clientjar>

	<appear name="ejb_sam_twobean"/>
    </package>
  </target>

Example 2:
==========
	 <target name="package">
    <package>

	   <ejb-jar descriptor="ejb_depEcmp20_pkey_ejb.xml" name="ejb_depEcmp20_pkey">
	<support includes="com/sun/ts/tests/assembly/util/wrapper/CMP20WrapperEJB.class, com/sun/ts/tests/assembly/util/dbsupport/CompoundPK.class"/>
	     </ejb-jar>

	   <clientjar descriptor="ejb_depEcmp20_pkey_client.xml" name="ejb_depEcmp20_pkey"
	 mainclass="com.sun.ts.tests.ejb.ee.deploy.entity.cmp20.pkey.Client">
	    <support	includes="com/sun/ts/tests/assembly/util/dbsupport/CompoundPK.class"/>
	   </clientjar>

	   <appear name="ejb_depEcmp20_pkey"/>
    </package>
  </target>
