appear

Description:

appear is designed for building application archives.  By default it includes ${ts.home}/dist/${pkg.dir}/*.jar and ${ts.home}/dist/${pkg.dir}/*.war.  If certain component archives need to be explicitly included or excluded, use support sub-element.

Parameters:

Attribute

Description

Required

name

Controls the naming convention used to name generated EJB jars. 

Yes

Nested Elements

The <appear> task provides the following nested elements.

support

The <support> nested element is used to explicitly include or exclude certain component archives.

Examples

Example 1:
==========
  <appear name="foo">
	   <support includes="${pkg.dir}/*.jar, ${pkg.dir}/*.war"/>
  </appear>
  <!-- support element in the above example is optionally.	 appear	will behavior the
		same even if support is	not used -->


Example 2:
==========
	 <target name="package">
    <package>
      <ejb-jar descriptor="ejb_sam_Hello_ejb.xml" name="ejb_sam_Hello">
	</ejb-jar>
	  <clientjar	descriptor="ejb_sam_Hello_client.xml" name="ejb_sam_Hello"
	      mainclass="com.sun.ts.tests.samples.ejb.ee.simpleHello.HelloClient">
      </clientjar>
	   <appear name="ejb_sam_Hello"/>
    </package>
  </target>
