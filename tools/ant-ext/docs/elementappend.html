elementappend

It is designed to modify each element within a pattern so that a generic pattern can be used in multiple forms.
 

Parameters:


Attribute
Description
Required
original
original value of the pattern
Yes
property the name of the property to store the modified pattern
Yes
prefix the string that will be added in front of the original pattern either one or both prefix and suffix must be specified
suffix the string that will be appended after the original pattern either one or both prefix and suffix must be specified

 

EXamples:

<target name="build.all" depends="tslib">
      <property name="all.test.dir" value="signaturetest, interop, integration, ejb, common, jdbc, connector, jms, jaxp, xa, servlet, jsp, appclient, javamail, rmiiiop, assembly, jta, samples"/>
      <elementappend original="${all.test.dir}" prefix="com/sun/ts/tests" suffix="**/*.java" property="all.java.file"/>
      <echo message="Compiling:${all.java.file}"/>
      <javac fork="true" srcdir="${ts.home}/src" 
            destdir="${class.dir}" 
            includes="${all.java.file}"
            memoryInitialSize="100m"
            memoryMaximumSize="200m"
            debug="on" 
            deprecation="on" >
            <classpath>
                <pathelement path="${ts.classpath}:${local.classes}" />
            </classpath>
      </javac>
      <dosubdirs srcdir="${ts.home}/src/com/sun/ts/tests" todo="package" includes="${all.test.dir}"/>
    </target>
