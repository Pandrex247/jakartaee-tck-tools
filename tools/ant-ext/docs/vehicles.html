vehicles

Description:

This task is designed to support building of application archives for various vehicles.  It queries the harness to determine the required vehicle set for the current test directory.

Parameters:


Attribute
Description
Required
mainclass
Client class
Yes
name
Controls the naming convention used to name generated archives 
Yes
autocheckexcludes files not to be automatically included into archives
No

Nested Elements

The webwar task provides the following nested elements.

support

The <support> nested element is used to supply additional classes (files) to be included in the generated wars. The <support> element is a FileSet, so it can either reference a fileset declared elsewhere or it can be defined in-place with the appropriate <include> and <exclude> nested elements. The files in the support fileset are added into the generated WEB jar in the same relative location as their location within the support fileset. Note that when webwar generates more than one jar file, the support files are added to each one.
Content (for war's that need content)

The <content> nested element is used to supply web content (e.g., *.html, *.jsp, *.tld, etc) to be included in the generated wars.  You do not need to specify content for war's that do not need content.  Use multiple content subelements if the war takes content from multiple locations.  Note that the content will be put into war's only.

Examples

Example 1:
==========
  <target name="package">
    <package>
      <vehicles name="transport" mainclass="com.sun.ts.tests.samples.javamail.ee.transport.send_Test">
          <content dir="contentRoot"/>
  <content dir="." includes="*.jsp"/>
        </vehicles>
    </package>
  </target>
