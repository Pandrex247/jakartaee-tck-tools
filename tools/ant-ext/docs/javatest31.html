Quickstart Javatest 3.1

Batch mode:

  • Run batch as usual (in a test directory, tsant runclient)
  • By default test report will be generated in /tmp/JTreport at the end of test run, and test.jtr files are  generated in work directory /tmp/JTwork.  You can specify a custom report.dir or work.dir via system properties, for example:
    • tsant runclient -Dreport.dir="/home/bh123456/JTreport"
    • tsant runclient -Dwork.dir="/home/bh123456/JTwork"
  • To skip test report, run runclient target with -Dreport.dir="".  (tsant runclient -Dreport.dir="") . 

GUI mode:

  • Start GUI (tsant gui)
  • If there is no previous test run record,  you will be prompted to create work directory.  Enter /tmp/JTwork
  • Click the start icon on the tool bar to start running ALL tests in the testsuite.  Start icon is the third item on the toolbar.
  • To run selected tests, right click the test folder, and select from context menu.
  • To generate test report, open report menu, and specify report directory (/tmp/JTreport).  Report will NOT be automatically generated.
  • Most of the time, you do not need to run configuration editor.  Default values have been set for all required interview questions:

Default
env file ts.home/bin/ts.jte
env ts_unix or ts_win32

specify exclude list Yes
which exclude list ts.home/bin/ts.jtx

  • If you need to create a new interview file (jti file), open configuration menu => new.  No defaults are used when you create a new interview file.  Alternatively, you can choose configuration menu => edit, and you will be editing the default interview settintgs.
  • Click stop icon on the toolbar to stop test run.
  • To view test run report, open report menu => view.  Javatest has a built-in browser to view test report.
